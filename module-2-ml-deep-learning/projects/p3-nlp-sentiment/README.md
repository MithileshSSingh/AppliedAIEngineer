# P3: NLP Sentiment Analysis

## Objective
Build a sentiment analysis system using multiple approaches: classical ML (TF-IDF), fine-tuned transformer, and pretrained pipelines. Compare performance and analyze errors.

## Dataset
Synthetic product reviews generated by `src/generate_data.py`:
- 3,000 product reviews with sentiment labels (positive/negative/neutral)
- Varying review lengths and complexity
- Real-world noise: typos, informal language, sarcasm

## Architecture
```
Approach 1: TF-IDF + Classical ML (LogReg, SVM, NaiveBayes)
Approach 2: HuggingFace pretrained sentiment pipeline
Approach 3: Fine-tuned DistilBERT (stretch goal)
```

## Key Skills
- Text preprocessing and vectorization
- Classical NLP with sklearn
- HuggingFace transformers and pipelines
- Model comparison across paradigms
- Error analysis for NLP

## Suggested Approach
1. Generate and explore review data
2. Preprocess text (clean, tokenize)
3. Build TF-IDF + classical ML baselines
4. Run HuggingFace pretrained pipeline
5. Compare all approaches
6. Error analysis: where do models disagree?
7. (Stretch) Fine-tune DistilBERT

## How to Run
```bash
python src/generate_data.py
jupyter lab notebook.ipynb
```
