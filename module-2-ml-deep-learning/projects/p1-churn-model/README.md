# P1: Customer Churn Prediction Model

## Objective
Build an end-to-end binary classification pipeline to predict customer churn. Practice the full ML workflow from EDA through model selection, evaluation, and interpretability.

## Dataset
Synthetic telecom churn dataset generated by `src/generate_data.py`:
- 5,000 customers with 15+ features
- ~20% churn rate (realistic imbalance)
- Mix of numeric, categorical, and derived features

## Architecture
```
Raw Data → EDA → Feature Engineering → Model Training → Evaluation → SHAP Analysis
```

## Key Skills
- Binary classification with imbalanced classes
- Multiple model comparison (LogReg, RF, XGBoost, LightGBM)
- Proper evaluation: precision, recall, F1, ROC-AUC, PR-AUC
- Threshold tuning for business objectives
- SHAP interpretability (Week 6)

## Suggested Approach

### Week 5: Model Building
1. Generate and explore the data
2. Feature engineering and preprocessing pipeline
3. Train baseline (LogisticRegression)
4. Train tree-based models (RF, XGBoost, LightGBM)
5. Compare models with cross-validation
6. Tune the best model
7. Evaluate on test set with full metrics

### Week 6: Interpretability & Business Impact
1. SHAP analysis on best model
2. Feature importance deep dive
3. Business cost analysis (cost of false positives vs false negatives)
4. Threshold optimization
5. Final recommendations

## How to Run
```bash
python src/generate_data.py   # Creates data/churn_data.csv
jupyter lab notebook.ipynb     # Main analysis
```
