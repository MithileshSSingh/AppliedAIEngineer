# P1: Cloud ML Data Pipeline

## Objective
Build an end-to-end cloud data pipeline that ingests raw customer data, validates quality, preprocesses features, and stores training-ready datasets — using S3 patterns (local simulator with optional real AWS).

## Dataset
- **Source:** Synthetic churn data generated by `src/generate_data.py`
- **Size:** 8,000 customers, 18 features
- **Quality issues:** ~2% nulls, ~1% duplicates (intentional for pipeline practice)
- **Alternative:** Any CSV dataset — the pipeline is dataset-agnostic

## Architecture
```
Raw CSV  →  S3 Upload  →  Lambda Trigger  →  Validate  →  Preprocess  →  Parquet to S3
  (raw/)                   (simulated)       (quality)    (clean)        (processed/)
```

## Key Skills
- boto3 S3 operations (put, get, list, head)
- Data validation and quality checks
- Lambda handler design (event-driven)
- CSV → Parquet conversion for ML
- Pipeline orchestration with status tracking
- Error handling and logging

## Deliverables
1. `notebook.ipynb` — Main pipeline walkthrough
2. `src/generate_data.py` — Synthetic data generation
3. `src/s3_pipeline.py` — LocalS3Simulator, validation, preprocessing, pipeline runner
